## System Architecture

```
RSS Sources (8 core feeds)
    ↓
research_monitor.py (fetches articles)
    ↓
articles_raw.json (raw article data)
    ↓
curate_articles.py (AI evaluation via Claude)
    ↓
digest.txt (your personalized reading list)
```

## Day 4-5: AI Curation Integration

Two options for Week 1:

### Option A: Manual Curation (Fastest)
1. Run `research_monitor.py` to get articles
2. Copy `articles_raw.json` content
3. Paste into Claude.ai with curation prompt from `curate_articles.py`
4. Claude returns scored/curated articles
5. Create your digest manually

**Time:** ~20 minutes/day, fully functional by Day 4

### Option B: API Integration (More Automation)
1. Get API key: https://console.anthropic.com/
2. Add to `curate_articles.py` (code structure already in place)
3. Run automated pipeline
4. Iterate on prompts

**Time:** ~2-3 hours to set up, then 5 min/day automated

**Recommendation:** Start with Option A for Week 1. You'll learn faster by manually seeing what Claude produces. Automate in Week 2 once you've refined the curation logic.

## Customization Points

**In research_monitor.py:**
- `hours_lookback`: Change from 24 to fetch older/newer articles
- `RSS_SOURCES`: Add/remove sources as you test

**In curate_articles.py:**
- `THEME_DEFINITIONS`: Refine what each theme means
- `JJ_CONTEXT`: Add/update your priorities
- `CURATION_PROMPT_TEMPLATE`: Adjust scoring criteria
- `min_score`: Filter digest to show only scores ≥ X

## Troubleshooting

**No articles fetched:**
- Check internet connection
- Verify RSS URLs still work (sites sometimes change feed URLs)
- Try increasing `hours_lookback` to 48 or 72

**RSS parse errors:**
- Some feeds are malformed - skip them for now
- Add to Phase 2 list

**Too many articles:**
- Reduce `hours_lookback` 
- Remove high-volume sources temporarily
- Increase `min_score` in digest generation
